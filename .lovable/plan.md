
# モノモン v3.0 全体リデザイン計画

規模が大きい（画面7・部品12・約4,000行）ため、**5フェーズに分けて順に納品**します。各フェーズ単独でも一貫性が壊れないように区切っています。1フェーズ完了ごとに実機確認していただき、次に進みます。

## 判断基準（すべての改修に共通）
「これは実物のモノをより大切にしたくなるか？」→ No なら削除／作り直し。

---

## Phase 1 ─ 情報設計の整理（土台）

v3.0 が明確に「表示しない」と定めている要素を UI から外し、以後の再設計の土台をつくる。

- **削除**：レア度（RarityStars／★表示）／種族カタログモード／収集率（○/50体）／統計グリッド／連番・XP的な表示
- **図鑑（zukan.tsx 1008行）を「思い出」に改名**し、二層構造（Species/Album）を廃止 → 単一の時系列日記へ簡素化
- ボトムナビは既に3タブなので維持（ホーム／カメラ／思い出）
- なかよし度メーター（0-100の数値バー）は v3.0 のゲーム機構除去に該当 → **数値と進捗バーを外し、表情と一言だけ残す**（内部ロジックは保持）
- コピー全面見直し：「発見数」「集める」「レア」「Loading」→ 詩的な表現へ

## Phase 2 ─ カメラを「出会い」へ

- 「撮り直す／モノモンを探す」等の操作語を、選択肢としては残しつつ弱く（小さく灰色）
- DiscoveryReveal を全面書き換え：
  - 「モノモンを探してるよ…」等の説明文 → 無言＋光と粒子のみ
  - 段階名（SCAN/GATHER/SILHOUETTE...）はそのまま、テキストを削る
  - 最終メッセージは **「出会えた。」** の一語だけ（名前は少し遅れて小さく）
  - コンフェッティ・派手なSEを撤去、静かな鈴の音1つに
- エラー画面（GentleError）も詩的な短文に：「今日はうまく会えなかったみたい」等

## Phase 3 ─ 思い出（旧・図鑑）を絵本ページへ

- 一覧：グリッドではなく **縦スクロールの絵本ページ**。1画面=1体、大きな写真＋日付＋モノ名＋小さなモノモン＋一行の詩
- 詳細画面：現行の複雑なカード／星／なかよし度バー／削除・共有ボタン群を撤去 → 写真主役の1画面
- 「NEW!」バッジは静かなドット1つに
- 削除・お気に入り・共有は控えめなメニューに集約（プライマリではない）

## Phase 4 ─ ホームとキャラ表現

- ホーム：現行の骨格を維持しつつ、モノモンは **一部だけ見せる**（下から覗く／端から見る）レイアウトへ調整
- home-companion 画像を「覗く/眠る」構図に差し替え
- MonomonArt に「peek」表示モード追加（下1/3だけ描画）
- アニメーション統一：呼吸・瞬き・浮遊の3種のみ、CSSで一元管理

## Phase 5 ─ 全体トーンの仕上げ

- 設定画面：アイコン過多を整理、章立ての静かなリストへ
- IntroOverlay：ゲーム的な説明ではなく「ようこそ」の一枚絵に
- 全画面の余白・タイポ・影を Design System v3.0 のトークンに統一
  - 背景 `#FAF8F3` を全画面に
  - 影は `0 10px 30px -14px rgba(60,45,25,0.35)` 系に統一
  - フォントウェイトは最大 600 まで（現行 800 は撤去）
- 全コピーを句読点なし・短文に統一（既存メモリの text-style に準拠）

---

## 保持するもの（触らない）
- ルーティング構造（TanStack Start / 3タブ）
- AI認識ロジック（monomon-ai.functions.ts / classification.ts / species.ts）
- なかよし度の内部データ（表示だけ静める）
- iOS/Capacitor 設定、Stripe応援、AutoUpdater
- Supabase連携、認証、公開URL

## 削除する具体的ファイル・機能
- `src/lib/rarity.ts` の UI 参照（ファイルは残しつつ import を消す）
- zukan.tsx 内の `RarityStars`, `Mode="species"`, カタログ表示
- FriendshipMeter の数値バー（コンポーネントは残し、表示モードで切替）

## 技術的メモ
- 大規模な zukan.tsx は一度**新規ファイルで書き直し**、旧ファイルを置換（差分編集では追いつかないため）
- 画像資産は必要に応じて imagegen で生成（Phase 4）
- 破壊的変更のためデプロイ後は AutoUpdater が既存ユーザーの画面を自動更新

---

## 進め方
1. まず **Phase 1** に着手（1つのターンで完了予定）
2. 実機で確認いただき、フィードバックがあれば反映
3. 問題なければ **Phase 2** へ、以降同様

**確認事項**：この5フェーズ構成と削除方針で進めてよろしいですか？特に以下は最終確認させてください：
- レア度・種族カタログ・収集率・数値の「なかよし度」表示 → **すべて撤去**でOK？
- 応援（Stripe）機能 → 静かな導線に整えて**残す**でOK？
